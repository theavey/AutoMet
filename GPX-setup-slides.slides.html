Setting up calculations for parallel tempering¶

Thomas Heavey, John Porco, David Coker¶

Proposed work for an NSF funding opportunity with the SAIL team

Why are we interested¶

Current process¶

  • The Porco group will bring us a new set of molecules in which they are interested.
  • I will go through this setup process that is pretty formulaic but getting through the steps can take a bit.
  • I will then run a simulation.
  • Following the simulation, in collaboration with the experimentalists, we will plot some distributions and pull out some snapshots of the simulation for viewing or further computations.

This could be much more efficient¶

  • My expertise isn't really used for most of this except for in dealing with errors (I've mostly caused), and seeing how efficiently I can get it done.
  • There are likely other systems for which they would like similar simulations if the barrier to entry were lower.
  • Basic analyses could likely be automated and done without needed me if there were a simplied interface.

Current interface¶

  • I have managed to get 95% of the work into Python so I don't have to leave that environment.
  • This does include several shell calls, to access other programs and tools.
  • All plotting of data is done with my tools in Python, but there are some follow-up calculations we do that I mostly do from a shell.
  • Ideally, we wouldn't expect Python or shell knowledge for basic usage.

    kwargs = {'template': str(template),
              'topology': str(top),
              'structure': str(gro_equil),
              'index': str(index),
              'scaling_exponent': scaling_exponent,
              'start_temp': start_temp,
              'maxwarn': maxwarn}
    with f_pt.as_cwd():
        try:
            os.mkdir('TOPO')
        except FileExistsError:
            if skip_existing:
                print(f'Skipping {key} because it seems to '
                      'already be done.\nMoving on...')
                continue
        with cd('TOPO'):
            print(f'Now in {os.getcwd()}\nAttempting to compile TPRs...')
            pts.compile_tprs(**kwargs)
            print('Done compiling. Moving on...')
        print(f'Now in {os.getcwd()}\nWriting submission script...')
        with sub_templ.open(mode='r') as templ_f, \
          open('gromacs-start-job.sub', 'w') as sub_s:
            [sub_s.write(l.format(**d_sub_templ)) for l in templ_f]
        print('Done.\nNow submitting job...')
        cl = ['qsub', 'gromacs-start-job.sub']
        proc = subprocess.run(cl, 
                              stdout=subprocess.PIPE,
                              stderr=subprocess.STDOUT,
                              universal_newlines=True)
        if proc.returncode == 0:
            output = proc.stdout
            jobs.append(re.search('[0-9].+\)', output).group(0))
            print(output, '\nDone.\nMoving to next...')
        else:
            print('\n\n'+5*'!!!---'+'\n')
            print(f'Error with calling qsub on {key}')
            print('Command line input was', cl)
            print('Check input and try again manually.'
                  '\nMoving to next anyway...')
            failed_procs.append(proc)
print('-----Done-----\nSummary of jobs submitted:')
for job in jobs:
    print(job)

Concluding notes¶

My thoughts¶

  • There is a lot that I've already simplified into Python functions and classes.
  • There is a lot more that is a very short step away from that.
  • A few steps would need some thought and effort on how to interface with other programs or to remake their functionality.
  • If we wanted a simple Python notebook interface for most of this, I could do that with 6 months to a year with no or few other projects.
  • Some of the other stuff would be outside of my current skillset.
  • Getting this to a user-friendly GUI is something in which I have no experience.

Components that could significantly benefit from a GUI¶

  • Selection of variables for histogramming
    • It could be nice to select two atoms then easily get a plot showing frequency versus distance.
    • If this were implemented, this could also be used to setup other types of calculations, sampling along the selected distances.
  • Selecting a region of such a plot
    • We often pull out structures based on such plots, but it could be easier.
    • The correspondence between a plot and the structures that contribute to that region could be easily found.
